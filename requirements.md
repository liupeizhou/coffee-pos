# 产品需求文档 (Requirements)

## 1. 产品概述

### 产品名称
咖啡店点单系统 (Coffee POS)

### 产品类型
桌面端应用 (Electron + React)

### 核心定位
面向咖啡店前台/吧台的点单结算系统，支持产品管理、订单处理、员工管理、换班交接和数据报表功能。

### 目标用户
- 咖啡店前台/吧台员工
- 咖啡店管理员/店长

---

## 2. 目标用户与痛点

### 目标用户

| 角色 | 使用场景 |
|------|---------|
| 前台员工 | 快速下单、收款结算、交接班 |
| 管理员 | 产品管理、查看报表、数据导出 |

### 核心痛点

| 痛点 | 描述 |
|------|------|
| 手动记账效率低 | 传统手工记录订单容易出错、统计困难 |
| 库存/销售不清 | 无法实时了解产品销量和库存情况 |
| 班次交接混乱 | 缺乏规范的交接班流程和账目核对 |
| 数据分析困难 | 销售数据分散，难以进行同比环比分析 |

---

## 3. 功能需求

### 3.1 产品管理
- [ ] 分类管理：新增/编辑/删除产品分类
- [ ] 产品管理：新增/编辑/删除产品（名称、价格、图片、分类）
- [ ] 产品选项：规格（大小）、温度（冷/热）、做法选项
- [ ] 产品做法：不同做法的价格调整

### 3.2 订单管理
- [ ] 点单功能：从分类选择产品，添加到购物车
- [ ] 购物车：数量调整、删除单品、备注
- [ ] 结算：选择支付方式、计算找零
- [ ] 会员折扣：支持会员卡折扣
- [ ] 历史订单：查看历史订单记录

### 3.3 员工管理
- [ ] 登录登出：工号+密码登录
- [ ] 角色区分：管理员 / 普通员工
- [ ] 权限控制：仅管理员可清除数据

### 3.4 换班系统
- [ ] 上班打卡：记录上班时间、交接金额
- [ ] 下班交班：记录下班时间、当班营业额
- [ ] 换班记录：查看历史换班记录

### 3.5 报表系统
- [ ] 每日销售：当日订单数、营业额、优惠金额
- [ ] 产品销量：按产品统计销售数量、销售额排行
- [ ] 同比环比：昨日/上月同期对比
- [ ] Excel导出：日报表、月报表导出

### 3.6 数据管理
- [ ] 系统设置：店铺名称、会员折扣、支付方式
- [ ] 数据清除：清除订单和班次数据（管理员操作）

---

## 4. 非功能需求

### 性能
- 页面加载时间 < 2秒
- 订单创建响应 < 500ms

### 可靠性
- 异常日志记录
- 数据持久化（SQLite本地存储）

### 可用性
- 简洁直观的UI设计
- 快捷键支持（ESC关闭弹窗）

### 兼容性
- macOS (当前主要目标)
- Windows (后续支持)
